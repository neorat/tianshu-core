# 银行完整账户体系设计（五层账户模型）

## 来源
- 链接：对话粘贴
- 来源：ChatGPT
- 时间：2026-04-21

## 问题

银行核心系统至少存在 5 类账户（主账户、子账户、虚拟账户、总账账户、内部户），很多团队只理解客户银行卡账户，导致系统设计不完整。如何建立完整的账户体系分层模型？

## 结论

**银行账户体系由五类账户构成，各司其职、缺一不可：**

| 账户类型 | 核心职责 | 本质 |
|---------|---------|------|
| **主账户** | 客户资金归属主体 | 客户资金明细账 |
| **子账户** | 资金隔离（真钱独立） | 客户资金明细账（细分） |
| **虚拟账户** | 识别/分账/核算（无真钱） | 统计维度 |
| **内部户** | 银行自有运营资金 | 银行资金明细账 |
| **总账账户(GL)** | 会计报表与监管报送 | 财务汇总账 |

## 关键论据

### 1. 账户体系总体结构

```text
┌─────────────────────┐
│     总账账户(GL)     │
│ 会计报表 / 财务科目   │
└─────────┬───────────┘
          │ 汇总映射
  ┌───────┴───────────────┐
  │                       │
┌─▼──────────────┐  ┌────▼───────────┐
│  客户账户体系    │  │  内部账户体系    │
│  (External)    │  │  (Internal)    │
└───────┬────────┘  └────┬───────────┘
        │                │
  ┌─────┴─────┐    ┌─────┴──────────┐
  │           │    │ 清算户/手续费户  │
┌─▼───┐  ┌───▼────┐│ 待处理账户等    │
│主账户│  │虚拟账户 │└────────────────┘
└──┬──┘  └────────┘
   │
┌──▼─────────────────────┐
│ 子账户（真钱隔离账户）   │
└────────────────────────┘
```

### 2. 五类账户详解

#### 主账户（Main Account）

客户视角的核心资金账户，是客户资金关系主体。

- 典型：储蓄账户、对公结算账户、钱包主账户、信用卡主账户
- 职责：客户资金归属主体、对外展示余额、开户销户主体、对账主体
- 核心字段：`account_id, customer_id, currency, status, available_balance, ledger_balance`

#### 子账户（Sub Account）

挂在主账户下的真实资金账户，余额独立。

- 典型场景：多钱包、保证金、冻结资金、产品分仓、多币种分户
- 关键规则：`主账户余额 = Σ 子账户余额`（可选模型），或主账户仅作关系主体不存余额
- 示例：张三钱包 → 活期钱包 / 理财冻结户 / 营销返现户

#### 虚拟账户（Virtual Account）

无真实资金，仅用于识别、分账、核算。

- 典型场景 1：企业收款虚拟账号（母账号 6222xxxx → VA10001 订单A / VA10002 订单B）
- 典型场景 2：商户门店分账（主账户 100 万 → 上海门店 60 万 / 深圳门店 40 万）
- 特征：不直接持真钱、可映射主账户、承载统计维度

#### 总账账户（GL Account）

银行会计账务账户，用于财务报表与监管报送，是科目级账户而非客户账户。

- 示例：100101 库存现金 / 201001 活期存款 / 220301 应付利息 / 601001 利息收入
- 与客户账户关系：客户账户是明细账，GL 是总账
- 客户存款 +1000 → 借：库存现金，贷：活期存款

#### 内部户（Internal Account）

银行自身运营资金账户，不属于客户。

- 常见内部户：手续费收入户、清算过渡户、待处理账户、差错账户、备付金户、渠道结算户、营销补贴户、资金池账户
- **任何银行系统若没有内部户，很多交易无法闭环**
- 示例：手续费扣款 → 借：客户账户 10，贷：手续费收入户 10

### 3. 标准分层模型

```text
L1 客户层：Customer
L2 账户层：Main / Sub / Internal / Virtual
L3 账务层：Journal Entry
L4 会计层：GL Subject
L5 报表层：监管 / 财务报表
```

### 4. 推荐数据模型

#### account 表（统一账户主表）

```
account_id, account_no, account_type, owner_type, owner_id,
parent_account_id, currency, status
```

#### account_type 枚举

```
MAIN, SUB_REAL, VIRTUAL, INTERNAL, GL_CONTROL(optional)
```

#### gl_subject 表

```
subject_code, subject_name, subject_type, normal_balance
```

#### account_subject_rel

```
account_id, subject_code
```

### 5. 典型错误（避坑）

| 错误 | 后果 |
|------|------|
| 客户账户 = GL 账户 | 无法扩展 |
| 虚拟账户直接出金 | 资金风险极大 |
| 内部户散落各系统 | 应统一账务中心管理 |
| 主子账户关系硬编码 | 未来多层账户树受限 |

### 6. 推荐服务拆分

```text
account-service          → 主账户/子账户/内部户生命周期
virtual-account-service  → 虚拟账号生成映射
ledger-service           → 分录记账
gl-service               → 科目汇总总账
recon-service            → 对账系统
```

### 7. 分阶段演进

| 阶段 | 内容 |
|------|------|
| Phase 1 | 主账户 + 内部户 + 记账引擎 |
| Phase 2 | 子账户 + 冻结户 + 虚拟账户 |
| Phase 3 | GL 系统 + 多法人账套 + 监管报表 |
