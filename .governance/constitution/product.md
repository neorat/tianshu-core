# 产品目标与愿景

## 产品愿景

天枢银行核心系统群（TianshuBankingPlatform）：以 DDD 驱动的企业级银行核心系统，
覆盖"管钱、算钱、记钱、防风险"四大能力域，为银行业务提供完整的数字化基座。

## 核心用户

| 用户角色 | 描述 | 核心需求 |
|---------|------|---------|
| 银行业务人员 | 柜面/渠道操作人员，处理日常交易、开户、转账等 | 交易受理快速准确，状态可追溯 |
| 信贷审批人员 | 负责贷款进件审批、签约、贷后管理 | 流程清晰可编排，风控决策有据可依 |
| 风控/合规人员 | 实时监控交易风险，管理规则与黑白名单 | 实时拦截、规则可配置、审计可追溯 |
| 运维/对账人员 | 日终对账、差异处理、清结算核对 | 内外部数据一致性保障，差异自动发现 |
| 产品经理 | 定义金融产品参数（利率、费率、期限等） | 产品参数灵活配置，上下架可控 |

## 系统全景

系统群包含 13 个有界上下文，每个上下文对应一个微服务：

| 分类 | 系统 | 一句话职责 |
|------|------|-----------|
| 核心 | 交易系统（Transaction） | 即时金融交易的统一入口和状态机 |
| 核心 | 支付系统（Payment） | 支付指令的编排、路由与执行 |
| 核心 | 账务系统（Accounting） | 复式记账，管理所有账户余额 |
| 核心 | 清结算系统（Clearing） | 交易后的清分、轧差和结算 |
| 核心 | 风控系统（RiskControl） | 实时风险评估与拦截 |
| 核心 | 客户中心（CustomerCenter） | 客户全生命周期管理（KYC/画像） |
| 核心 | 产品中心（ProductCenter） | 金融产品定义与参数配置 |
| 核心 | 额度中心（CreditLimit） | 授信额度的分配、占用与释放 |
| 核心 | 信贷流程管理（LoanProcess） | 信贷业务全流程编排（指挥官） |
| 核心 | 信贷核心（LoanCore） | 还款计划、利息计算等信贷逻辑（计算引擎） |
| 支撑 | 对账系统（Reconciliation） | 内外部数据核对与差异处理 |
| 支撑 | 卡系统（CardSystem） | 银行卡全生命周期管理 |
| 通用 | 用户中心（UserCenter） | 登录认证、权限管理 |

详细系统定义见 `domain/contexts/system-landscape.yaml`，
系统间关系见 `domain/contexts/context-map.yaml`，
全景导读见 `domain/contexts/system-landscape-guide.md`。

## 成功标准

- [ ] 13 个有界上下文的领域边界清晰，职责无交叉
- [ ] 资金流转主线（交易→支付→账务）端到端可用
- [ ] 信贷业务线（进件→审批→放款→还款）端到端可用
- [ ] 复式记账借贷平衡不变量在所有场景下成立
- [ ] 系统间集成模式（ACL/OHS/PL 等）落地到代码

## 非目标（明确不做的事）

- 前端渠道层（App/Web/小程序）— 本系统群只提供后端服务
- 外部监管报送 — 不在核心系统群范围内
- 数据仓库/BI 分析 — 由独立的数据平台承载
- 网银/手机银行 UI — 属于渠道层，不在本项目范围

## 约束

- 合规约束：银行业务需满足复式记账借贷平衡、账务流水不可篡改等金融级要求
- 架构约束：1 个有界上下文 = 1 个微服务，跨上下文通过领域事件或 RPC 通信
- 技术约束：详见 `constitution/technical.md`
