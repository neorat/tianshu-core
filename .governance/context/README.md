# 迭代上下文模块（可选）

> 本模块为 AI 提供"短期记忆"，解决跨会话状态丢失问题。

## 解决的问题

大模型无记忆。如果没有持久化的上下文，你在第一天跟 AI 达成的方案，在第三天让 AI 改 Bug 时它就会全忘掉。

本模块通过静态文件持久化"当前重点"和"迭代上下文"，让 AI 每次会话开始时能瞬间恢复记忆。

## 目录结构

```
context/
├── README.md          # 本文件
├── current-focus.md   # 当前开发焦点（每次会话必读）
├── backlog.md         # 待办事项（按优先级排序）
└── sprint-goal.md     # 当前迭代目标
```

## 文件职责

| 文件 | 更新频率 | AI 读取时机 | 内容 |
|------|---------|-----------|------|
| `current-focus.md` | 每天或每次焦点切换 | 每次会话开始 | 当前在做什么、优先级、阻塞项 |
| `backlog.md` | 任务完成或新增时 | 需要了解待办时 | 全部待办，用 `- [ ]` / `- [x]` 追踪 |
| `sprint-goal.md` | 每个迭代开始时 | 需要了解迭代目标时 | 本迭代要交付什么 |

## 使用方式

### 与 steering 集成

在 Kiro 中，可以通过 steering 文件让 AI 自动读取上下文：

```markdown
---
inclusion: auto
---

每次会话开始时，先读取迭代上下文了解当前工作重点：
#[[file:../../.governance/context/current-focus.md]]
```

### 与 specs 的关系

- `specs/` 记录功能级别的规范（做什么、怎么做）
- `context/` 记录迭代级别的状态（现在在做哪个、做到哪了）
- 两者互补：specs 是静态的设计文档，context 是动态的进度追踪

### 维护纪律

- 任务完成后，在 `backlog.md` 中将 `- [ ]` 改为 `- [x]`
- 焦点切换时，更新 `current-focus.md`
- 迭代结束时，归档旧的 `sprint-goal.md` 内容，写入新目标

## 何时启用本模块

- 项目开发周期超过一周
- 需要跨多次 AI 会话保持上下文连续性
- 有明确的迭代节奏（周迭代、双周迭代等）
- 多人协作，需要共享当前进度
