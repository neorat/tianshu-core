# 产品目标与愿景

## 产品愿景

天枢（Tianshu）是一套 AI 驱动的银行核心系统群工程化实践平台。目标是通过 AI 工程方法论，完成银行核心业务系统的领域建模、架构设计和可投产级代码落地，同时验证和完善"AI + 项目治理"的工作流与最佳实践。

## 双重目标

1. **业务目标**：构建可投产的银行核心系统群，覆盖交易、支付、账务、清结算、风控等核心领域
2. **方法论目标**：通过实战验证 AI 辅助的工程治理体系（领域建模 → 架构决策 → TDD → 代码审查），形成可复制的实践范式

## 核心用户

| 用户角色 | 描述 | 核心需求 |
|---------|------|---------|
| 架构师 | 负责系统群整体架构设计和领域边界划分 | 清晰的上下文映射、系统间契约定义、架构决策记录 |
| 后端开发者 | 基于领域模型实现各子系统的业务逻辑 | 规范的 DDD 工程骨架、明确的编码约束、AI 辅助开发流程 |
| 业务分析师 | 梳理银行业务流程和规则 | 统一语言字典、业务模型可视化、需求到代码的可追溯性 |

## 系统群范围

| 系统 | 分类 | 一句话职责 |
|------|------|-----------|
| 交易系统 | 核心域 | 承载所有金融交易的生命周期管理 |
| 支付系统 | 核心域 | 支付指令的路由、执行与状态管理 |
| 账务系统 | 核心域 | 复式记账、账户余额管理与账务流水 |
| 清结算系统 | 核心域 | 交易的清分、轧差与资金结算 |
| 对账系统 | 支撑域 | 内外部数据核对与差异处理 |
| 风控系统 | 核心域 | 实时/准实时风险评估与决策 |
| 用户中心 | 通用域 | 用户身份认证、授权与会话管理 |
| 客户中心 | 核心域 | 客户信息管理、KYC、客户画像 |
| 产品中心 | 核心域 | 金融产品定义、参数配置与生命周期 |
| 额度中心 | 核心域 | 授信额度的分配、占用、释放与管控 |
| 卡系统 | 支撑域 | 银行卡的发卡、绑卡、卡状态管理 |
| 信贷流程管理 | 核心域 | 信贷业务全流程编排（进件→审批→放款→还款） |
| 信贷核心 | 核心域 | 信贷合同、还款计划、利息计算等核心逻辑 |

## 成功标准

- [ ] 完成所有子系统的有界上下文定义和上下文映射
- [ ] 每个子系统的核心聚合根完成领域建模（YAML 契约）
- [ ] ArchUnit 测试覆盖所有分层约束
- [ ] 至少 3 个核心系统完成可运行的 MVP
- [ ] AI 工程治理工作流经过实战验证并形成文档

## 非目标（明确不做的事）

- 不做前端/移动端工程（专注后端微服务）
- 不做通用技术框架（framework 独立仓库维护）
- 不做运维部署平台（专注业务系统本身）
- 不追求一次性全部落地——渐进式，按优先级逐个系统推进

## 约束

- 技术栈锁定 Java 21 + Spring Boot 4.0+ + Maven
- 严格遵循 DDD 分层架构（interfaces → application → domain ← infrastructure）
- 1 个有界上下文 = 1 个微服务 = 1 个 Maven 多模块工程
- 金融级数据一致性要求：账务必须复式记账，资金操作必须幂等
- 所有系统间交互通过明确的契约（RPC 接口 / 领域事件），禁止直接数据库访问
